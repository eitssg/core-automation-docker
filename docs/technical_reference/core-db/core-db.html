core-db

Core-autmation provides a database schema that is used to store the state of the application. This schema is used to store the state of the application and is used to provide a history of changes to the application infrastucture deployments.

The database schema can be found here…

Base Models

core_db.actions

All table Actions share a CRUD interface and this Model class defines the functions for Create, Read, Update, Delete functions.

core_db.response

Response module for the core_db package.

core_db.dbhelper

A set of helper functions that write and update item records status values in the DB

core_db.constants

Constants for filed names and keys for the Core-DB databae tables

core_db.exceptions

Provides a set of Exceptions that the DB engine uses to provide feedback of operations

FACTS Database

core_db.facter

The Facter module contains the function and classes to create a VIEW on the Registry.

core_db.registry

The Registry maintains FACTS for clients, portfolios, zones, and apps

Deployments Database

core_db.item

The Item module contains the function and classes to create a CRUD interface to core-automation-items DynamoDB table.

Event Log Database

core_db.event

The Event module contains the function and classes to create a CRUD interface to core-automation-events DynamoDB table.
