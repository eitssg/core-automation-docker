Apps Table

The app Registry is a lists of deployments within a specific portfolio. A deployment is targeted at a specific zone

Table name: core-automation-apps

Attribute

Type

Description

ClientPortfolio (hash)

str

The f”{client}:{portfolo}” string

AppRegEx (range)

str

A regular expresstion matching the the string f”{app}-{branch}-{build}”

Example: “^(.+)-(.+)-(.+)$”

Zone

str

The Zone where this app is to be deployed

Region

str

The region name sepcified in the Zone definition

map

A set of name/value mappings that are to be applied to the resource tags

Items Table

The items table provides a dataabase of all Portfolios, Apps, Branches, Builds, and Components that have been deployed to the Landing Zones.

The following data models are in stored the Items table:

  • Portfolio Schema

  • App Schema

  • Branch Schema

  • Build Schema

  • Component Schema
